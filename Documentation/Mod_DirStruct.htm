Mod Directory Structure

When creating a MOD it is recommended you follow the data structure laid out in this section of the document, the structure can be changed but for the sake of this document we are going to assume you follow these guidelines.

In the install directory there is a MODs folder.  It is recommended you create a folder directly under this folder that corresponds with the name of whatever MOD you are creating, then in that folder create a data folder. IMPORTANT: The name of your mod can no more than 15 characters long You can see an example of this in the SampleMOD folder.  This is important so the structure of your built MOD is correct and able to be read properly by the game after you have built the MOD.  After this is done the directories created under data are up to the user wants to organize your data. Required Files:

There is only one required file for MOD building and that is MOD.xml.  This file is the file that should include any other files you are adding/editing for your MOD.  This is the file that the batch file will try to build and directly corresponds to the name of the manifest file that will be created by the build process.  This manifest is also hard coded in the game code so renaming it will cause your MOD not to work. For full details on the Mod.xml file, please read the Mod.XML Page.

The only other file that is hard coded as such is the MOD.str file.  This file is optional and only needed if you are trying to add or modify text in the game.

MOD.str Structure

An example of how an entry in the MOD.str file is below:

NAME:Orca
"Orca Fighter"
END

To replace an existing String entry, simply create a new entry in your Mod.str file with the same name as the entry you would like to replace. You can create new entries simply by adding them to the Str file, and reference them from within the .XML files, or Worldbuilder scripts. If you forget to add a string to the STR file, it will show up in the game as MISSING: folowed by the name of the string entry.

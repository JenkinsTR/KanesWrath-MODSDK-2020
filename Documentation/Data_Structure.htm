Data Structure

When creating a MOD it is recommended you follow the data structure laid out in this section of the document, the structure can be changed but for the sake of this document we are going to assume you follow these guidelines.

In the install directory there is a MODs folder.  It is recommended you create a folder directly under this folder that corresponds with the name of whatever MOD you are creating, then in that folder create a data folder.  You can see an example of this in the SampleMOD folder.  This is important so the structure of your built MOD is correct and able to be read properly by the game after you have built the MOD.  After this is done the directories created under data are up to the user wants to organize your data.

Required Files:

There is only one required file for MOD building and that is MOD.xml.  This file is the file that should include any other files you are adding/editing for your MOD.  This is the file that the batch file will try to build and directly corresponds to the name of the manifest file that will be created by the build process.  This manifest is also hard coded in the game code so renaming it will cause your MOD not to work.

The only other file that is hard coded as such is the MOD.str file.  This file is optional and only needed if you are trying to add or modify text in the game.

MOD.XML:

This file is the main file that will be compiled by the BinaryAssetBuilderApplication. This file should include all files that you are MODding/editing.  If it is not included in this file it will not be built.  There are two files that should always be included as a reference when creating a MOD.  These are static.xml and global.xml to represent the two main data streams the game loads, here is an example from the Sample MOD:

<Include type="reference" source="DATA:static.xml" />
<Include type="reference" source="DATA:global.xml" />

See the section on the BinaryAssetBuilder.exe.config for an explanation of the DATA: value. 

Art Files:

The Sample MOD includes artwork used for the Elven Warrior.  These assets can be found in the EU, and HC sub folders of the main Art folder.  Please note that the use of ART: in an include statement will automatically create a sub folder  consisting of the first two letters in the art file you are including.  For example:

<Include
         type="all"
         source="ART:EULorWar_SKN.w3x"
      />

This file will be searched for in the Art\EU folder.  Changing the location that ART: looks in will be covered in the next section.  Also any art files you work on will need to be located in this Art folder not as a sub folder of your MOD directory, unless you specifically change the location of the ART: folder.

BinaryAssetBuilder.exe.config

This file allows you to modify the behavior of the BinaryAssetBuilder program.  What this section will cover however is only the search paths for the program.  In the file you will find the section:

<assetbuilder
            serializedType="BinaryAssetBuilder.Settings,BinaryAssetBuilder.Core"
          schema="..\schemas\xsd\CnC3Types.xsd"                            
          dataRoot=".."
          defaultArtPaths=".\Art"
          defaultAudioPaths=".\Audio;"
          defaultDataPaths=".;.\Cnc3Xml;.\Mods"     
      > 

dataRoot – Is the root folder for all data.  This location is currently the main install directory of the MOD SDK.
defaulArtPaths – Default search path(s) prefixed with ART:.  Please note that using this value will append a 2 letter subfolder consisting of the first two letters of your art asset.
defaultAudioPaths- Search path(s) for paths prefixed with AUDIO:.
defaultDataPaths- Default search path(s) for all data files included using the DATA: value.  As currently setup it will search the current directory first, then the MODs folder, then the CnC3Xml folder.

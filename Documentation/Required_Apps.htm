Required Applications

The MOD SDK requires some applications be installed before being able to build the included Sample MOD or your created MOD.  The .NET Framework is required to be able to run the BinaryAssetBuilder application used to build the XML data.  The other two applications ANTRL, and Mvp.xml are not required but recommended.  Some of our existing xml files will not compile without these applications. 

  • .NET Framework v2.0 – Required to run the BinaryAssetBuilder application.
  • Visual C++ 2005 SP1 – Required to run the BinaryAssetBuilder application.
  • ANTLR v2.77 – Required when using the <Define> tag.
  • Mvp.xml v2.0 – Required when using xi:include statements.

Downloading and Installing the .NET Framework v2.0:

Please visit Microsoft’s website for information on downloading and installing the .NET Framework v2.0.

Downloading and Installing Visual C++ 2005 SP1:

The Installer for Visual C++ 2005 SP1 can be found at this Microsoft page. Follow the instructions on the page to install this. It is also compatible with Windows Vista, and will install properly, despite it not being listed on the web page.

Downloading and Installing ANTLR:

You can find ANTLR at http://www.antlr2.org.  Download version 2.7.7 (Note later versions may work but this is the version we tested our sample MOD with.).  In the folder you installed ANTLR to find the lib folder and copy the following 2 DLL files (antlr.runtime.dll, and antlr.astframe.dll) to the tools folder located inside the location you installed the MOD SDK. Default Path: "c:\program files\Command & Conquer 3 Tiberium Wars\MOD SDK".

Downloading and Installing Mvp.Xml:

You can find Mvp.Xml at http://www.codeplex.com/MVPXML.  Download version 2.0 (Note later versions may work but this is the version we used during development.).  Extract the zip file to a folder on your computer and then copy the Mvp.Xml.dll file to the tools folder located inside the location you installed the MOD SDK. Default Path: "c:\program files\Command & Conquer 3 Tiberium Wars\MOD SDK".
